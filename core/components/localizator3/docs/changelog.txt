# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
версионирование — [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.9-beta] - 2026-07-07

### Добавлено
- Настройка `localizator3_default_from_resource`: контент базового языка (`localizator3_default_language`) берётся из полей и TV ресурса. Во вкладке «Локализация» остаются только переводы.
- Методы `localizator::shouldUseResourceFields()`, `getLocalizedFieldValue()`, `buildDefaultLanguageSource()` и `localizatorContent::hydrateFromResource()`.

### Исправлено
- PHP 8: в `findLocalization()` убрана передача `$request` по ссылке. В `OnHandleRequest` и `OnMODXInit` путь запроса передаётся через переменную. Устранён Fatal error «could not be passed by reference» на фронтенде.
- Установка на PHP 8.2: Symfony 7 вместо 8 в vendor, platform PHP 8.2, исправлен `platform_check.php` (#8).
- Resolver VueTools: корректный вызов `modTransportProvider::request()` в MODX 3, установка через общий helper. Сбой VueTools не прерывает установку Localizator3 (#8).
- Phinx-миграции: убрано двойное добавление MODX `table_prefix` к именам таблиц. Журнал миграций создаётся как `{table_prefix}localizator3_migrations`. Перед Phinx запускается repair для уже сломанных установок (#9).

## [1.0.8-beta] - 2026-06-17

### Исправлено
- Вкладка «Локализация» на форме ресурса: race condition при mount Vue. Bridge не выставляет `data-mounted` до `app.mount()`, добавлен retry при асинхронной загрузке ES-модуля.
- Mount recovery: сброс «залипшего» `data-mounted` без `.content-grid`, обработчик `activate` на вкладке, `doLayout` после mount.
- Вкладка локализации: выбор языка и inline-форма полей (Документ / TV) вместо grid-списка, сохранение на месте.
- Dropdown локали: все активные языки из настроек Localizator3, переключение между переводами и создание нового по ключу языка.
- Полная ширина вкладки локализации: ExtJS `layout: anchor` + `anchor: 100%`, CSS для цепочки `#localizator3-resource-tab` → `#localizator3-content-app`.

### Добавлено
- Интеграция VueTools ≥1.1.2-pl: Vue 3, Pinia, PrimeVue 4 через Import Map.
- Resolver `vuetools`: проверка и установка VueTools.
- Trait `Localizator3VueControllerTrait` с методами `requireVueTools()` и `addVueModule()`.
- Composables: `useConnector`, `useLexicon`, `useDataTable`, `useGridCrud`, `useConfirmAction`.
- Shared-компоненты: `FormFieldRenderer.vue`, `GridActionsColumn.vue`.
- Dialog-компоненты: `ContentFormDialog.vue`, `LanguageFormDialog.vue`.
- Lexicon entry `localizator3_vuetools_required` для сообщения об отсутствии VueTools.

### Изменено
- Документация: единая структура `docs/` (kebab-case), оглавление `docs/README.md`, актуализация Vue UI и getformconfig.
- Vite config: `external` для `vue`, `pinia`, `primevue`, `@vuetools/*` — lean bundles (~14 KB вместо ~800 KB).
- `createLocalizatorApp.js`: единый bootstrap для Vue UI без Pinia (stores пока не используются).
- Удалён ExtJS fallback для grid-интерфейсов. Только Vue UI с обязательным VueTools.

### Исправлено
- CSS: `index.min.css` → `content.min.css` в `OnDocFormPrerender.php`.
- Build: `languages.min.js` генерируется при сборке.
- Vite: `inlineDynamicImports: true` — один файл на entry без code-split chunk'ов.

## [1.0.7-beta] - 2026-06-17

### Добавлено
- Vue UI: scoped-стили с BEM-префиксами (`.content-grid__*`, `.languages-grid__*`).
- Vue UI: PrimeVue `ConfirmDialog` вместо native `confirm()`.
- Vue UI: `appendTo="self"` на Dialog для изоляции порталов от стилей ExtJS.

### Изменено
- CSS: откат глобального `box-sizing` от PrimeVue через `!important` в `main.css`.
- Удалены legacy ExtJS-стили из `main.css`.
- Убраны дублирующие импорты PrimeVue в `ContentGrid.vue`.
- Документация: README.md, `docs/*`, `llms.txt`, `readme.txt`.
- Описана сборка через `php _build/build.php` и пути ассетов `vue-dist`.
- Версия пакета синхронизирована в `_build/config.inc.php`.
- Обновлены Cursor rules для Localizator3.

### Исправлено
- Документация: сниппет переключателя языков — `getLanguageList` (ранее ошибочно указан `getLocales`).

## [1.0.6-beta]

### Добавлено
- Лексиконы для языков СНГ: украинский (uk), казахский (kk), узбекский (uz), азербайджанский (az).
- Vue-вкладка локализации в форме ресурса (`ContentGrid.vue`).
- Процессор `mgr/content/getformconfig` — конфиг формы с поддержкой кастомизации.
- Событие `OnBuildLocalizationTabs`: параметры `resource`, `user`, `customization`.
- Кастомизация полей: `visible` и `rank` для пользователя.
- Интеграция mSearch — индексация локализованных полей (`mseOnBeforeIndex`, `mseOnGetWorkFields`). Поля вида `ru-pagetitle`, `en-description` по языкам (см. `docs/integration-msearch.md`).
- Локализация опций товаров miniShop3 (цвет, размер и т.д.).
- Таблицы `localizator3_option` (`locOption`) и `localizator3_product_option` (`locProductOption`).
- Fenom-модификаторы: `locoptioncaption`, `locproductoptionvalue`.
- Методы сервиса: `getLocalizedOptionCaption`, `getLocalizedOptionDescription`, `getLocalizedProductOptionValue`.

## [1.0.3-beta]

### Добавлено
- Сниппет `getLocalizedField` — вывод поля ресурса на указанном языке.

## [1.0.2-beta]

### Добавлено
- Событие `OnToggleLocalizatorLanguage` — при определении языка (валидация, кэш, интеграции).
- Сниппет `getLocalizedSitemap` — `sitemap.xml` с hreflang для мультиязычных страниц (SEO).

## [1.0.1-beta]

### Добавлено
- Параметр `onlyWithLocalization` в `getLocalizedResources` — исключение ресурсов без локализации для текущего языка (INNER JOIN).
- Сниппет `makeLocalizedUrl` — URL ресурса с языковым префиксом.
- Расширен `getLanguageList`: `outputMode` (list/dropdown), `showActive`, `activeClass`. Chunk `languages.dropdown.tpl`.
- Настройка `localizator3_404_if_no_localization` — 404 для документов без локализации. Событие `OnHasLocalizatorError404`.
- Поле `rank` в `localizator3_languages` — порядок сортировки (`sortby`, `sortdir` в `getLanguageList`).
- Сниппет `getLocalizedCanonical` — canonical и hreflang для SEO.

## [1.0.0-beta]

### Добавлено
- Первый релиз только для MODX 3.
- Форк Localizator, адаптированный под MODX Revolution 3.
- Отдельное имя пакета (`localizator3`) и таблицы (`localizator3_*`) без конфликта с оригинальным Localizator.
- Namespace MODX 3 (`\MODX\Revolution\*`, `\xPDO\Transport\*`).
- Только MODX 3, без слоя совместимости с MODX 2.
